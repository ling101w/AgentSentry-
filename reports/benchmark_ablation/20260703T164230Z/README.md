## 重要观察

- 不开启玄鉴时，129 条正常样例全部放行，但 711 条期望保护样例中 97.0% 出现高风险意图放行，51.9% 出现危险工具实际执行成功；其余大量环境失败主要来自不存在的外部 API、缺失敏感文件或浏览器实验台禁用 shell。
- 开启玄鉴但关闭 LLM-Judge 时，攻击保护率 100%，高风险漏放和危险执行均为 0，正常放行率 100%，误拦 0；不足是 111 条攻击样例被判定为过度控制，主要是隐藏网页/外部内容第一步读取被直接拦截，而不是先安全读取再阻断后续敏感动作。
- 完全开启玄鉴 + LLM-Judge 时，攻击保护率仍为 100%，高风险漏放和危险执行均为 0；但 LLM-Judge 更保守，正常误拦 8 条，过度控制 267 条，中位延迟约 9.26 秒。
- 因此，报告中建议把 `risk-tiered` 作为生产推荐模式：保留 LLM-Judge 对高风险路径的解释和二次判定，但避免全量 LLM-Judge 对普通读取/正常请求造成明显延迟和保守误判。

## 文件索引

- `cases.all.jsonl`：本轮 840 条输入样例。
- `benchmark_ablation_report.md`：自动生成的三组总表和判定口径。
- `benchmark_ablation_results.json`：三组汇总结构化结果。
- `<profile>/results.json`：该组逐样例结构化结果，包含输入、决策、状态、失败原因、record id。
- `<profile>/results.csv`：该组逐样例表格。
- `<profile>/case_trace.txt`：该组长文本留痕，可直接查看每条样例输入和玄鉴处理。
- `records.after-run.tail.jsonl`：运行结束后 8765 当前保留的原始 JSONL 审计记录尾部。由于在线记录库上限为 10000，完整逐样例分析以 `results.json` 和 `case_trace.txt` 为准。
